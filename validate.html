Validate

Click Frameworkの特徴の一つは、ValidationがUIと一体化されている点だとおもいます。
各データタイプ毎に例えば 、 このSampleで使用した、IntegerFieldや、DoubleField、LongFIeld、NumberField、DateField、EmailField、RegixFieldなどExtrasには各種含まれており、また自分で拡張するのも容易です。またDataType毎になっているので、設定する際や読み取る際もString型に変換する必要がありません。
JavaScriptとも連携しており、たとえばIntegerFieldに数字以外を入力しても、無視されると共に、最大値や最小値のCheckが自動的に行われ、それに違反した場合はエラー表示がフォームのなかに自動的に表示されますので、わざわざコードを書く必要がなく、大変簡単に処理できます。
それでは、画面とそれを設定するJAVA コードを見てみましょう

package org.seasar.s2click.example.page.validate;

import net.sf.click.control.Form;
import net.sf.click.control.Submit;
import net.sf.click.extras.control.EmailField;
import net.sf.click.extras.control.IntegerField;

public class ValidatePage extends org.seasar.s2click.example.page.BorderPage {
	
	private Form form = new Form("form");
	public ValidatePage(){
		addControl(form);
		IntegerField intfield = new IntegerField("int","Integer Test 最大10");
		intfield.setMaxValue(10);
		form.add(intfield);
		form.add(new EmailField("email","EMAIL ADDRESS"));
		form.add(new Submit("submit"));
	}
}
上記の様にIntegerFieldに最大値 １０の設定が容易にできます。ここに１０以上の値を入れると、下記の様にエラー表示されます
エラーのフィールドの背景色が黄色に変わり、エラーメッセージが表示されます
